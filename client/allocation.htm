Chat room allocation

So you are trying to talk on Chattrr, and you find that you aren't talking on the URL at which you have pointed your browser. Why not?

Chattrr tries to intelligently figure out where you want to be talking, and puts you in a chat room where some people are. It won't just yoink one out of the blue though.

The first thing it does is to go and see if there enough people chatting where you are, eg http://aurl.com/path/to/file.htm. If there is, then great, you'll be talking to everyone else there! If there isn't though, it will allocate you to the chat room at http://aurl.com/path/to/, and if there aren't enough there, then it will try http://aurl.com/path and http://aurl.com. If there aren't people talking at any of these then it puts you at the global chat room at http://chattrr.net.

Conversely, if you are pointing at a URL and there is no one there, but there are lots of people at the parent directory, you probably won't want to be there since you won't be able to get a word in edgeways. In this case, you will want to start your own chat room — enough people will be there soon enough! In this case chattrr will make a new board for you and now keep allocating people to the more specific URL instead of the parent.

If you are talking at a top level URL and it is too busy, then there's not much chattrr will do — yet. If we find it's necessary, we can create subchannels on each URL.

But I really want to be talking to myself at http://elanorribgy.com!  Ok, that's fine. We handle that. Just type in /force and you can be all alone!
